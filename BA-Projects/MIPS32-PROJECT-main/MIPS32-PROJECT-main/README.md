# MIPS32 Project

## Overview
This project was completed during my Bachelor’s degree studies for the Computer Architecture course.

The project was written in MIPS32 Assembly. It reads a 12-digit code word and checks whether there is an error in the code word using parity-check logic.

## Project Purpose
The purpose of this project was to practise low-level programming and understand how data is stored, accessed, and processed at assembly level.

## Technologies Used
- MIPS32 Assembly
- MARS / SPIM simulator
- Computer architecture concepts
- Registers and memory addressing
- Branching and loops

## Main Features
- Reads a 12-digit code word from user input
- Stores input values in memory
- Uses arrays for parity-check positions
- Checks parity groups for possible errors
- Displays whether the code word has an error or not

## What I Learned
- How to write programs in MIPS32 Assembly
- How registers are used in low-level programming
- How to work with memory and arrays in Assembly
- How loops and branches work at machine level
- How parity-check logic can be implemented

## Academic Context
This project was completed as part of my BA studies in the Computer Architecture course.
