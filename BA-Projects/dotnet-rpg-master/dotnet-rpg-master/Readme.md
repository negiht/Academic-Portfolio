# .NET RPG Tutorial Project

## Overview
This project was completed during my internship as part of a tutorial provided by the company to help me start learning backend and frontend development concepts.

The project is a basic RPG character API built with ASP.NET Core. It focuses on creating backend endpoints, working with controllers and services, using DTOs, and returning structured API responses.

## Project Purpose
The purpose of this project was to practise the fundamentals of backend development and understand how a web API is structured using .NET.

## Technologies Used
- C#
- ASP.NET Core
- .NET
- REST API
- Swagger
- Object-oriented programming
- Backend development concepts

## Main Features
- Character model for RPG characters
- RPG classes such as Knight, Mage, and Cleric
- API endpoint to get all characters
- API endpoint to get a character by ID
- API endpoint to add a new character
- DTOs for adding and returning character data
- Service layer for character logic
- Swagger interface for testing the API

## Project Structure
The project includes:

- `Controllers/` — API controllers
- `Models/` — application models such as Character and RPG class
- `Dtos/` — data transfer objects
- `Services/` — service logic for handling characters
- `Program.cs` — application configuration and service registration
- `appsettings.json` — application settings
